# 修改总结 - 打包下载功能改进（2026-03-27）

## 新增功能
实现项目文档打包下载功能，支持按周期/文档类型/子目录组织文件，并自动添加层级序号。

## 后端修改

### 1. project_doc_manager/app/services/task_service.py
新增 `start_download_package_task()` 方法：
- 实现异步打包任务，支持进度跟踪（0-100%）
- 按照 `cycles` 列表顺序处理周期
- 按照 `required_docs` 顺序处理文档类型
- 支持子目录分组和层级序号生成

**编号规则：**
- **无子目录**：`周期.序号`（如 `1.1`, `7.1`, `7.2`）- 2位
- **有子目录**：`周期.文档类型.子目录.文件序号`（如 `3.3.1.1`, `3.3.1.2`）- 4位
- **规则**：只有子目录才增加位数，文档类型层不增加

**目录结构示例：**
```
1.项目立项/
├── 1.1 文件1.pdf
├── 1.2 文件2.pdf
└── 1.3 文件3.pdf

7.系统开发测试/
└── 7.17 项目用户操作手册/
    ├── 7.1 手册1.pdf
    └── 7.2 手册2.pdf

3.项目准备/
└── 3.3 项目合同/
    └── 3.3.1 大唐南京发电厂/
        ├── 3.3.1.1 合同.pdf
        └── 3.3.1.2 协议.pdf
```

**辅助函数：**
- `clean_filename()`: 清理文件名，去掉非中文字符前缀，添加序号
- `clean_name_prefix()`: 去掉名称前面的非中文字符

### 2. project_doc_manager/app/routes/task_routes.py
新增两个路由：
- `POST /api/tasks/download-package`: 启动打包任务
- `GET /api/tasks/download/<task_id>`: 下载打包好的文件

### 3. project_doc_manager/app/routes/documents/files.py
修改 `select_files()` 函数：
- 使用前端传来的原始文件名（`file_info.get('name')`）保存到 `filename` 和 `original_filename` 字段
- 修复临时文件名（如 `unknown_20260325_200131.pdf`）显示问题

## 前端修改

### 4. project_doc_manager/static/js/modules/project.js
新增/修改函数：
- `handleDownloadPackage()`: 触发打包任务，显示进度模态框
- `showDownloadProgressModal()`: 显示打包进度
- `updatePackageProgress()`: 更新进度条
- `closeDownloadProgressModal()`: 关闭模态框

### 5. project_doc_manager/templates/index.html
新增打包进度模态框：
- 显示进度条（0-100%）
- 显示当前状态信息
- 完成后显示手动下载链接（3秒后自动关闭）

## 关键修复

### 文件名显示问题
**问题：** 手动上传到 tmp 目录的文件显示为 `unknown_20260325_200131.pdf` 而不是原始文件名

**原因：** `select_files()` 保存的是磁盘上的临时文件名

**修复：** 
1. 后端：`select_files()` 使用前端传来的 `file_info.get('name')` 作为原始文件名
2. 打包时：优先使用 `original_filename` 字段，其次是 `filename` 字段

### 周期顺序问题
**问题：** 打包后第一个显示的是"运维"而不是"项目立项"

**原因：** 直接遍历 `documents` 字典，顺序随机

**修复：** 使用 `project_config.get('cycles', [])` 列表确定正确顺序

### 文档顺序问题
**问题：** 文档类型顺序与页面显示不符

**原因：** 使用了 `sorted(doc_types)` 对文档类型排序

**修复：** 按照 `required_docs` 列表的顺序处理文档类型

## 数据格式说明

### uploaded_docs 条目格式
```json
{
  "cycle": "1、项目立项",
  "doc_name": "项目立项申请书",
  "filename": "关于集团公司智慧党建管理平台项目立项的请示.docx",
  "original_filename": "关于集团公司智慧党建管理平台项目立项的请示.docx",
  "file_path": "projects/智能党建项目/uploads/xxx/文件.docx",
  "directory": "/",
  "doc_id": "...",
  "upload_time": "..."
}
```

### 文件序号计算逻辑
- `file_seq` 在文档类型级别累加，确保同一文档类型的文件连续编号
- 子目录内的文件也在该计数器基础上继续累加

## 测试验证点
1. 打包下载按钮触发任务，显示进度条（0-100%）
2. 完成后自动下载 ZIP 文件
3. 解压后检查目录结构和文件名：
   - 周期顺序正确（1→2→3...）
   - 文档类型顺序与页面一致
   - 文件序号格式正确（无子目录2位，有子目录4位）
