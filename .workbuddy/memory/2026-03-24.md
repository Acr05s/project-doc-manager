## 项目目录结构

```
project_doc_manager 
 ├─app 
 │  ├─routes 
 │  │  └─__pycache__ 
 │  ├─tasks 
 │  │  └─__pycache__ 
 │  ├─utils 
 │  │  └─__pycache__ 
 │  └─__pycache__ 
 ├─projects/     # 用于存放项目的所有文件，项目的配置文件，文件匹配结果.json,/项目名/*.* 包括 该项目的访求需要.json，解压的文档压缩包的文件夹，项目打包的时候 直接打包/项目名 这个目录，后续就直接导入，改继续编辑，实现备份，分享项目。同时最后文档验收完成打包文档也是根据文件匹配结果.json实现打包文档。

```

## 目录结构调整

1. **创建顶级 projects/ 目录**：用于存放项目的所有文件，包括项目配置文件、文件匹配结果.json、项目的需求文件、解压的文档压缩包文件夹等。

2. **修改目录配置**：更新了 `app/utils/base.py` 文件中的 `DocumentConfig` 类，将项目基础文件夹路径从 `uploads/projects` 更改为顶级 `projects` 目录。

3. **确保目录自动创建**：利用现有的 `ensure_dir` 函数，当应用程序运行时会自动创建 `projects` 目录，确保目录结构的完整性。

4. **验证代码兼容性**：检查了 `project_manager.py` 和 `document_manager.py` 等核心模块，确认它们能够正确使用新的目录结构。

5. **测试目录结构**：运行应用程序，确认它能够正常启动，项目管理器能够加载项目，文档管理器能够正常初始化。
