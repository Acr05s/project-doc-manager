# 产品手册（使用说明）

## 1. 产品概览
本产品用于管理项目文档的采集、检查、归档、审批与导出，适用于多角色协作场景。

## 2. 角色与权限
- `admin`: 系统管理员，拥有全局管理权限。
- `pmo` / `pmo_leader`: 过程管理角色，可查看与管理跨项目事项。
- `project_admin`: 项目管理角色，管理所属组织项目。
- `contractor`: 普通用户，处理授权范围内项目文档。

## 3. 常用功能路径
### 3.1 创建与配置项目
1. 进入项目管理。
2. 新建项目并填写基本信息。
3. 导入需求模板或手工配置周期与文档要求。

### 3.2 上传与管理文档
1. 选择项目与周期。
2. 选择文档要求项。
3. 上传文件并填写属性（日期、签字、盖章等）。
4. 根据需要进行替换、删除、补充。

### 3.3 ZIP 批量导入
1. 上传 ZIP 包。
2. 执行文件匹配。
3. 确认匹配结果并导入。

### 3.4 归档与审批
1. 发起归档审批。
2. 审批人处理通过/驳回。
3. 查看审批历史。

### 3.5 定时报告任务管理
1. 打开“定时报告任务”。
2. 在管理列表中查看任务、执行次数与收件人分组。
3. 支持操作：新建、编辑、启用/停用、立即执行、删除。
4. 支持批量操作：批量启用、批量删除。

## 4. 升级与自动迁移
推荐使用：
```bash
./Daemon.sh upgrade
```

升级会自动完成：
1. 拉取代码。
2. 安装/更新依赖。
3. 自动迁移数据库与运行时数据文件。
4. 重启服务。

## 5. 运维常用命令
```bash
./Daemon.sh start
./Daemon.sh stop
./Daemon.sh restart
./Daemon.sh status
./Daemon.sh logs
./Daemon.sh upgrade
```

## 6. 排障建议
- 启动失败：先执行依赖安装，再查看日志。
- 升级失败：先确认 Git 状态与网络，再手工运行迁移脚本。
- 迁移失败：可根据自动备份文件回退，并重新执行迁移。

## 7. 安全与合规建议
- 首次部署后立即修改默认管理员密码。
- 定期备份 `data/`、`projects/`、`uploads/`。
- 避免在日志、注释、示例脚本中保留真实项目/人员信息。
