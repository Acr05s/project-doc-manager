# 部署和测试指南

## 📦 环境要求

- **Python**: 3.7+
- **操作系统**: Windows、Linux、macOS
- **浏览器**: 现代浏览器（Chrome、Firefox、Safari、Edge）

## ⚙️ 安装步骤

### 方法1：使用启动脚本（推荐）

#### Windows
```bash
run.bat
```

#### Linux/Mac
```bash
chmod +x run.sh
./run.sh
```

启动脚本会自动：
1. 检查Python环境
2. 创建虚拟环境（如果不存在）
3. 安装依赖包
4. 启动应用

### 方法2：手动安装

```bash
# 1. 创建虚拟环境
python -m venv venv

# 2. 激活虚拟环境
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动应用
python main.py
```

## 🚀 启动应用

启动成功后会看到：
```
 * Running on http://0.0.0.0:5000
 * Debug mode: on
```

打开浏览器访问：`http://localhost:5000`

## 🧪 测试步骤

### 1. 生成示例需求清单

```bash
python generate_sample_requirements.py
```

这会生成 `示例需求清单.xlsx` 文件。

### 2. 加载项目配置

1. 打开应用主页
2. 点击"加载项目配置"按钮
3. 选择 `示例需求清单.xlsx`
4. 点击"加载项目"

### 3. 上传测试文档

1. 在左侧选择一个项目周期
2. 在右侧找到一个文档卡片
3. 点击"查看/管理文档"按钮
4. 在弹窗中：
   - 选择一个PDF或图片文件
   - 填写文档日期和签字人
   - 点击"上传文档"

### 4. 测试签字识别（可选）

如果上传的是手写签字的图像，系统会自动识别签字并显示置信度。

### 5. 查看已上传文档

上传后，文档会立即显示在"已上传文档"列表中，包括：
- 文件名
- 上传时间
- 文件大小
- 是否有签字标记
- 识别结果标签

### 6. 生成报告

1. 点击右上角"生成报告"按钮
2. 查看各周期的完成进度
3. 点击"导出报告"下载为HTML文件

## 🐛 故障排除

### 问题1：无法启动Flask应用

**症状**: `ModuleNotFoundError: No module named 'flask'`

**解决方案**:
```bash
pip install flask
# 或重新安装所有依赖
pip install -r requirements.txt
```

### 问题2：端口5000被占用

**症状**: `Address already in use`

**解决方案**:
修改 `main.py` 最后一行：
```python
app.run(debug=True, host='0.0.0.0', port=5001)  # 改为5001
```

### 问题3：文件上传失败

**症状**: 上传后出现错误提示

**检查项**:
1. 上传目录是否有写入权限
2. 文件是否超过大小限制（50MB）
3. 文件格式是否被允许

### 问题4：签字识别不准确

**症状**: 签字识别误判

**改进方法**:
- 确保图片清晰
- 调整 `main.py` 中的检测阈值
- 将图片转换为高对比度

## 📊 性能优化建议

### 对于大型项目

1. **增加并发上传限制**
   - 修改 `upload.js` 中的 `maxConcurrent` 值

2. **优化文件存储**
   - 使用CDN存储上传的文件
   - 定期清理过期文件

3. **增强签字识别**
   - 使用GPU加速（opencv-python-headless-gpu）
   - 采用深度学习模型

### 对于多用户场景

1. **添加用户认证**
   - 集成Flask-Login
   - 添加权限管理

2. **使用数据库**
   - 替换内存存储
   - 使用SQLite/PostgreSQL

3. **实现日志系统**
   - 记录所有操作
   - 审计跟踪

## 📝 日志和调试

### 查看应用日志

应用会输出详细的日志信息，包括：
- 项目配置加载情况
- 文件上传过程
- 签字/盖章检测结果
- 错误和异常

### 启用详细日志

修改 `main.py` 的日志级别：
```python
logging.basicConfig(
    level=logging.DEBUG,  # 改为DEBUG获取更详细信息
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```

## 🔒 安全建议

1. **生产环境**
   - 不要使用 `debug=True`
   - 配置HTTPS
   - 添加认证和授权

2. **文件上传安全**
   - 验证文件类型和内容
   - 使用病毒扫描
   - 限制文件大小

3. **敏感信息保护**
   - 使用环境变量存储配置
   - 加密敏感数据
   - 限制日志输出
